Goal/s

The main objective of the website is to increase people's knowledge of the CoffeeContainer business by providing a single, easy-to-use platform where they can access information about menus, prices, specials, and cart rental. The homepage features a button that enables users to access the menu quickly, and the navigation bar prominently displays options to contact and learn more about the business. The website's goal is to facilitate easy communication between users and the CoffeeCan business, achieved through multiple links to its social media pages and a user-friendly form to help customers get in touch.

Developer Workflow

The first step is to install Wordpress on your own computer by Wordpress.org You must download WordPress after installation. WAMP (MAMP or XXAMP for MacOS and linux) and configure it. WAMP should be accessible in the bottom right of the screen after a successful installation. Go ahead and do the following:
  • Open WAMP options and select PhpMyAdmin
  • Just input the database name and click "create" to establish a new database with the name of your website.
  • Go to the MAMP directory on the system disc of your computer. Paste the Wordpress folder that has been unzipped into the htdocs folder.
  • To finish installation, enter localhost/"name of your wordpress file in the htdocs"

Adding Themes

  • Open Underscores
  • Be sure to use the "sassify option" when downloading, and give the file the name of the theme you want for your website.
  • Activate the theme by placing the file in the WordPress Themes folder and using the WordPress Admin.
  • Upon activation, you may upload the modifications and new theme to github.
Deployment You may use the command line to publish website updates to github, but if you're not comfortable with it, we highly recommend downloading Github Desktop as it has an easy to understand UI.If you are using the command line:
  • Use the web browser to access the wp-content folder. cd command
  • Use git status in the relevant place to view any changes.
  • Use git add (file path) to add any changed files
  • To commit changes and include a commit message, use git commit -m "commit message"
  • Use git push origin master to push changes to master on github
  • Use the git help command to get a list of useful information if you run into any problems or don't know how to use a command.
